# daily-trade-notes — 盘后总结精读与预期差映射

把用户每天的盘后总结(群聊+X博主+自己的操作计划)从"小作文"提炼成**可执行情报**:主线、点位、荐股、催化、风险,并用财报/预期差去伪存真。**这是情报整理,不是投资建议**。

## 数据来源(本地)
文件夹:`C:\Users\lixue\Desktop\st总结\`
- `每日总结.txt` — 按日期(如 6.15/6.16/6.17)累加的笔记,含大盘点位、主线、各博主荐股、个股操作计划。
- `微信图片_*.jpg` 等截图 — 多为X长文/研报/板块热度表(用 Read 读图)。

## 每日处理流程
1. 读 `每日总结.txt`,定位**最新日期段**(上次digest之后的新增内容);读新增的截图。
2. 结构化提炼,分四块:
   - **大盘点位逻辑**:支撑/突破/关键阈值(历史框架见下),机构仓位(中信加减仓)、反弹vs反转判断。
   - **主线/仓位方向**:今天强的板块、资金往哪转移。
   - **各路信源荐股**:按信源归类(见 references/sources.md),标注个股+板块+催化。
   - **个股操作计划**:用户/鹏睿等的买卖纪律(破5日/30日止损、做T、分批低吸)。
3. **去伪存真**(关键):
   - 多信源共振的**题材/板块**优先于单信源**个股**。
   - 喊单/导流(如"翻倍/跟车/进群/t.me/加微信")只记情绪、标注、不作依据、不点链接。
   - 把荐股映射到代码后,用 `expectation_gap_monitor.py`(C:\Users\lixue\Desktop\Glozin) 或 `china-quant-platforms` 算预期差/查财报二次过滤,区分"真业绩"vs"纯情绪/亏损陷阱"。
4. 写入 `references/digest/<YYYY-MM-DD>.md`(最新在前):四块要点 + 今日共振题材TOP + 去伪存真后的"值得看/需规避"清单 + 风险。
